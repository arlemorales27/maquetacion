1) Fundamentos: estructura mínima

Comprende la anatomía de un documento HTML5 y los metadatos esenciales.

Plantilla base

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Mi sitio</title>
  <meta name="description" content="Descripción breve de la página">
  <link rel="icon" href="/favicon.ico">
  <link rel="stylesheet" href="/styles.css">
</head>
<body>
  <header>...</header>
  <main>Contenido principal</main>
  <footer>&copy; 2025</footer>
  <script src="/app.js"></script>
</body>
</html>
  • <!DOCTYPE html> activa el modo estándar del navegador.
  • lang ayuda a accesibilidad y SEO.
  • viewport permite diseño responsivo en móviles.

Metadatos clave

  • <meta name="description">
  • <meta property="og:title/description/image"> (Open Graph)
  • <meta name="theme-color" content="#2b6cb0">
  • <link rel="canonical" href="https://ejemplo.com">
Incluye siempre título único y una descripción por página.

Definiciones de esta sección

  • <!DOCTYPE html>: declara el tipo de documento y activa el modo estándar.
  • <html lang>: raíz del documento; lang indica el idioma.
  • <head>: metadatos (no visibles) como SEO, estilos y scripts.
  • <meta>: metainformación (charset, viewport, descripción).
  • <title>: título que ves en la pestaña del navegador.
  • <link>: enlaza recursos externos (CSS, iconos).
  • <script>: carga/embebe JavaScript.
  • <body>: contenido visible de la página.
  • <header>/<main>/<footer>: áreas semánticas del layout.

2) Texto y tipografía

Titulares, énfasis, citas, código y tiempo/fecha.

Etiquetas comunes

<h1>Título 1</h1>
<h2>Título 2</h2>
<p>Un <strong>texto importante</strong> con <em>énfasis</em> y <mark>resaltado</mark>.</p>
<p>Atajos: presiona <kbd>Ctrl</kbd>+<kbd>S</kbd> para guardar.</p>
<p>Precio: <small>IVA incluido</small>. Agua: H<sub>2</sub>O. x<sup>2</sup>.</p>
<p><abbr title="Cascading Style Sheets">CSS</abbr> acelera el estilo.</p>
<time datetime="2025-08-21">21 de agosto de 2025</time>
<blockquote cite="https://ejemplo.com">Una cita breve.</blockquote>
<pre><code>console.log('hola');</code></pre>

Consejos

  • Usa un solo H1 por página.
  • Evita usar etiquetas solo para estilizar (usa CSS).
  • Incluye atributo cite en citas cuando corresponda.

Definiciones de esta sección

  • <h1..h6>: niveles jerárquicos de encabezado.
  • <p>: párrafo de texto.
  • <strong>/<em>/<mark>: importancia, énfasis y resaltado.
  • <kbd>: secuencias de teclas.
  • <small>: anotaciones/nota al pie.
  • <sub>/<sup>: subíndices/superíndices.
  • <abbr>: abreviatura con título explicativo.
  • <time datetime>: fecha/hora legible por máquinas.
  • <blockquote cite>: cita en bloque con fuente.
  • <pre>/<code>: texto preformateado y fragmentos de código.

3) Enlaces y navegación

Buenas prácticas para <a> y estructuras de navegación.

Enlaces seguros y accesibles

<nav aria-label="Navegación principal">
  <ul>
    <li><a href="#">Inicio</a></li>
    <li><a href="#productos">Productos</a></li>
    <li><a href="https://sitio-externo.com" target="_blank" rel="noopener noreferrer">Externo</a></li>
  </ul>
</nav>

<a href="mailto:contacto@ejemplo.com">Escríbenos</a>
<a href="tel:+573001112233">Llámanos</a>

Tips

  • rel="noopener noreferrer" en enlaces externos abre en nueva pestaña sin riesgos.
  • Usa textos de enlace descriptivos (evita “haz clic aquí”).

Definiciones de esta sección

  • <nav>: bloque de navegación.
  • <ul>/<li>: listas de elementos.
  • <a href>: enlace; target="_blank" abre nueva pestaña; rel="noopener noreferrer" mejora se

    4) Imágenes y multimedia

    Imágenes responsivas, figure/picture, audio/video con subtítulos.

    Imágenes semánticas

    <figure>
  <picture>
    <source srcset="paisaje.webp" type="image/webp">
    <img src="paisaje.jpg" alt="Paisaje montañoso al amanecer" width="800" height="533" loading="lazy">
  </picture>
  <figcaption>Figura 1. Imagen con <code><picture></code> y <code><figcaption></code>.</figcaption>
</figure>

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  Tu navegador no soporta el elemento audio.
</audio>

<video controls width="480" poster="poster.jpg">
  <source src="demo.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitulos.vtt" srclang="es" label="Español" default>
  Tu navegador no soporta el elemento video.
</video>

<iframe src="https://www.youtube.com/embed/dQw4w9WgXcQ" title="Video de ejemplo" allowfullscreen loading="lazy"></iframe>

    Buenas prácticas

    • Alt descriptivo en imágenes informativas.
    • Usa loading="lazy" para rendimiento.
    • Incluye pistas <track> (subtítulos) en video.

    7) Semántica y estructura

    Estructura con <header>, <main>, <section>, <article>, <aside> y <footer> + componentes interactivos.

    Layout semántico típico

    <header>
  <h1>Mi Blog</h1>
  <nav aria-label="Principal">...</nav>
</header>
<main>
  <article>
    <header><h2>Entrada</h2></header>
    <p>Contenido...</p>
    <footer><address>Por <a href="mailto:ana@blog.com">Ana</a></address></footer>
  </article>
  <aside>Contenido relacionado</aside>
</main>
<footer>© 2025</footer>

    Detalles / diálogo

    <details>
  <summary>Ver más</summary>
  <p>Contenido desplegable.</p>
</details>

<dialog id="dlg">
  <p>Diálogo nativo</p>
  <form method="dialog">
    <button>Cerrar</button>
  </form>
</dialog>
<button onclick="document.getElementById('dlg').showModal()">Abrir diálogo</button>

    Plantillas y componentes

    <template id="cardTpl">
  <article class="card p-3">
    <h4><slot name="titulo">Título</slot></h4>
    <p><slot name="contenido">Texto</slot></p>
  </article>
</template>

    Usa <template> para clonar fragmentos desde JS.

    8) Accesibilidad (a11y)

    Roles ARIA, etiquetas, foco y atajos.

    Ejemplos rápidos

    <button aria-pressed="false" aria-label="Abrir menú">
  <i class="fa-solid fa-bars" aria-hidden="true"></i>
</button>

<nav aria-label="Migas de pan">
  <ol>
    <li><a href="#">Inicio</a></li>
    <li aria-current="page">Artículo</li>
  </ol>
</nav>
    • Proporciona alternativas de texto y etiquetas.
    • Controla el foco y respeta Tab/Shift+Tab.

    Checklist mínimo

    • Contraste AA (4.5:1 texto normal).
    • Secuencia lógica de encabezados.
    • Estados visibles: foco/hover/activo.
