1) Definiciones precisas

Entiende el vocabulario clave antes de diseñar.

Sketch (boceto)

Dibujo rápido y preliminar que representa las características principales de un objeto o escena. Ideal para divergir y fallar barato.

Wireframe

Esquema estructural (baja/mediana fidelidad) que define jerarquía, layout y navegación.

Mockup

Diseño estático de alta fidelidad (colores, tipografías, componentes reales).

Prototipo

Versión interactiva que simula flujos y estados. Excelente para tests de usabilidad.

2) Sketches & Wireframes de baja fidelidad

Combina técnicas manuales y digitales para avanzar rápido sin atarte a la estética final.

Técnicas manuales

  • Crazy 8s: 8 ideas en 8 minutos.
  • Plantillas con rejilla (móvil/desktop) impresas.
  • Marcadores gruesos para pensar macro.

Técnicas digitales

  • FigJam / Figma para cajas y flechas rápidas.
  • Balsamiq para estética “sketch” y feedback temprano.
Principios UI/UX aplicados
  • Consistencia (componentes y estilos reutilizables)
  • Jerarquía visual (títulos 20–24, texto 16)
  • Espaciado 8‑pt (8/16/24/32)
  • Contraste AA para texto/íconos
  • Accesibilidad (roles, alt text)
  • Hick: menos opciones, decisiones más rápidas
  • Fitts: botones grandes y cercanos
  • Feedback y estados (hover, éxito, error)
  • Gestalt: proximidad y continuidad
  • Reglas de Nielsen: control del usuario, prevención de errores

4) Mockups de alta fidelidad

Aterriza la identidad visual (paleta, tipografía, componentes) listos para aprobación.

Checklist de mockup en Figma

  • Aplica tu sistema de diseño (botones, inputs, cards, espaciado).
  • Usa estilos de color y fuentes variables.
  • Presenta en device frames (móvil/desktop).
  • Exporta versiones para revisión (PDF/PNG) y gestiona comentarios.
Do
  • Contraste AA mínimo 4.5:1
  • Grid de 8‑pt
  • Estados: hover/activo/error
  • Iconografía consistente (línea/grosor)
Don't
  • Texto < 16px
  • Demasiadas variantes de botón
  • Color como único indicador
  • Padding inconsistente

Buenas prácticas

  • Define tareas medibles (p. ej., “crear cuenta en 3 pasos”).
  • Incluye estados (vacío, error, carga) y feedback.
  • Prueba con 5–8 usuarios; mide tiempo y tasa de éxito.

6) Tarea práctica guiada

  1. Sketch: 6 variantes de home móvil en 10 minutos (crazy 8s).
  2. Wireframe: elige 2 y pásalos a baja fidelidad (móvil + desktop).
  3. Mockup: aplica tu identidad visual en Figma.
  4. Prototipo: conecta 3 tareas clave (registro, crear ítem, confirmar).
  5. Itera: recoge feedback y repite.
